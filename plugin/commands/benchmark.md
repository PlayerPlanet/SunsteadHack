Benchmark a candidate agent with the **sunstead-bench** MCP server.

Arguments: `$ARGUMENTS`
- `agent_import` = `$1` (required) — `"module.path:Attr"` exposing `review(view) -> Decision`
  and a `name`. Reference adapter for the Arctal take-home agent:
  `bonds_instrument.candidates:DQAgentCandidate`.
- `materials_path` = `$2` (optional) — directory of bond CSVs; omit for the bundled sample.
- `--llm` (optional) — enable the candidate's LLM tier (needs `ANTHROPIC_API_KEY`).

Steps:
1. Call the `benchmark_agent` tool with `agent_import=$1`, `materials_path=$2` (if given),
   and `use_llm=true` iff `--llm` is present.
2. Report the scorecard plainly: the **region** (trustworthy ceiling), the overall
   **false-clear** (dangerous — acted past its edge), **over-ask** (wasted), and
   **justified-ask** (good) rates, and the per-drift-bin curve.

Doctrine (issue #28): the benchmark is generated by **re-derivation + planted labels**,
and the judge is **frozen and non-agentic** — the inputs that vary are the *agent* and the
*materials*, never the ruler. The interpretive layer (is the category truly right?) has no
answer key and is explicitly **not scored** — only the boundary behaviour is. Never grade a
run with a model; that re-enters the benchmark paradox.
