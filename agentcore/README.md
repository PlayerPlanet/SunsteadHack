# AgentCore project (bootstrap)

This directory holds the AgentCore CLI project config (`agentcore.json`,
`aws-targets.json`) that `agentcore deploy` (see `.github/workflows/deploy.yml`) reads.
Those files are **generated by the CLI**, not hand-authored — generate them once, wire
in our entrypoint, and commit. Steps (full context in `docs/deploy-agentcore.md`):

## 1. Install the CLI
```bash
npm install -g @aws/agentcore        # Node 20+
```

## 2. Scaffold an MCP project here
```bash
agentcore create --protocol MCP      # name it e.g. sunstead-control
```
This writes `agentcore/agentcore.json` + `agentcore/aws-targets.json` and an app dir.

## 3. Point it at our server
- **Entrypoint:** `cleanroom.control.server.runtime_mcp` (serves streamable-HTTP at
  `0.0.0.0:8000/mcp`, `stateless_http=True` — the AgentCore MCP contract).
- **Dependencies:** use `requirements-runtime.txt`, and ensure the `cleanroom` package
  is importable in the build (copy it into the app dir, or `pip install` the repo).
- **Env:** set `CLEANROOM_PG_APP_DSN` to the **non-superuser** `sunstead_app` DSN
  (ends `/sunstead_control` — see `sql/roles.sql`; provisioned already). Optionally
  `OAUTH_DEFAULT_ROLE`/`OAUTH_ROLE_CLAIM` to tune DB-role brokering from token claims.

## 4. Configure inbound auth (the IdP AgentCore validates against)
Provide the OIDC **Discovery URL** + **Client ID** at `agentcore create` (or via
`agentcore add`). Cognito setup snippet is in `docs/deploy-agentcore.md`.

## 5. Commit + deploy
Commit `agentcore/agentcore.json` + `agentcore/aws-targets.json`, then either
`agentcore deploy` locally or push to `main` with `DEPLOY_ENABLED=true`.

> Why no committed `agentcore.json` here yet: its schema is owned by the CLI and
> versioned with it; committing a hand-written one would risk drift. Generate it with
> the CLI version you deploy with.
